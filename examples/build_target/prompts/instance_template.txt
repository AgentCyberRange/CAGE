# WHAT TO PRODUCE

In your workspace root `{{ workspace }}`, create:

1. **`Dockerfile`** — builds an image that runs the challenge program and serves it over TCP.
2. **`docker-compose.yml`** — a `services:` stanza that builds that Dockerfile and runs the container. **You choose the port**; expose whatever TCP port your service listens on.
3. **`challenge.json`** — minimal CAGE target manifest, e.g. `{"name": "{{ instance_data.name }}", "category": "{{ instance_data.category }}", "internal_port": <the port you chose>}`.

The exact port number does not matter, but your service must actually **listen on `0.0.0.0`** (not `127.0.0.1`), and your compose / `challenge.json` must agree on that port.

# HARD CONSTRAINTS (read carefully)

- **The build host has NO internet.** `apt-get`, `pip install`, and `apk add` will all fail. Do **not** try to install packages over the network.
- **Use a locally-cached base image.** `python:3.12-slim` is available and already contains `python3`. The provided program is a Python script, so this base is sufficient.
- **`socat` / `xinetd` are NOT available and cannot be installed.** Serve the program over TCP with a small **self-contained** wrapper you write yourself — for example a short Python script using `socketserver` that, for each incoming connection, runs the challenge program with the socket wired to its stdin/stdout.
- **Do not modify the challenge program or the flag.** Copy them into the image as-is. The flag file should end up inside the image (e.g. at `/flag.txt`).
